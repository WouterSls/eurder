Iedere keer dat ik een testing dependency van maven afhaal kwam dit met een scope verwijzing
	<scope>test<scope>
echter werkt deze scope niet voor mijn project, het zegt telkens dat het deze packet niet vind.
Ik vermoed dat dit te maken heeft met het feit dat mijn src file in inteliJ na src verwijst naar
	java
	main
inplaats van
	main
	test

Ik heb dit telkens opgelost door de scope verwijzing weg te halen.
Ik weet wel dat dit niet best practice security is.
